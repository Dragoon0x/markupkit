React component detection

In React apps, MarkupKit reads the fiber tree and includes the component path. Depth adapts to your output level:

App›Dashboard›ActionBar›ExportButton
  • Compact — no React data
  • Standard — top-level component only
  • Detailed — filtered tree, framework internals hidden
  • Forensic — full tree including framework components

Installation

Get started with MarkupKit in your project.

Choose your setup

  • Just want annotations? → Basic Setup below (copy-paste to agent)
  • Using Claude Code? → Use the skill for auto-setup
  • Building custom tools? → Use the programmatic API

Install the package

terminal
npm install markupkit -D

Or use yarn, pnpm, or bun.

Add to your app

Add the component anywhere in your React app, ideally at the root level. The NODE_ENV check ensures it only loads in development.

layout.tsx
import { Markup } from 'markupkit'

function App() {
  return (
    <>
      <YourApp />
      {process.env.NODE_ENV === "development" &&
        <Markup />
      }
    </>
  )
}

Agent integration

Connect MarkupKit to any AI coding agent that supports MCP. This enables real-time annotation syncing and bidirectional communication.

1. Add the MCP server

terminal
npx add-mcp "npx -y markupkit-mcp server"

Auto-detects your installed agents (Claude Code, Cursor, Codex, Windsurf, etc.) and writes the correct config.

2. Verify setup

terminal
npx markupkit-mcp doctor

3. Connect the component

layout.tsx
<Markup
  endpoint="http://localhost:4747"
  onSessionCreated={(id) => console.log("Session:", id)}
/>
  • Local-first — works offline, syncs when server is available
  • Session continuity — rejoins the same session on page refresh
  • No duplicates — only new annotations are uploaded
  • Server authority — agent changes (resolve, dismiss) take precedence

Requirements

  • React 18+ — uses modern React features
  • Client-side only — requires DOM access
  • Zero dependencies — no runtime deps beyond React
  • ~7.3kb gzipped — tree-shakeable ESM and CJS

Security

MarkupKit runs in your browser and reads DOM content to generate feedback. By default, it does not send data anywhere — everything stays local until you manually copy.

  • No external requests — all processing is client-side by default
  • Local server only — the endpoint prop sends data to localhost only
  • No data collection — nothing is tracked or stored remotely
  • Dev-only — use the NODE_ENV check to exclude from production

API

Programmatic access for developers. MarkupKit exposes callbacks, types, an HTTP API, and real-time events for building custom integrations.

  • Sync annotations to a database or backend
  • Build analytics dashboards tracking feedback patterns
  • Create custom AI integrations (MCP servers, agent tools)
  • Pipe drawing gestures and detected elements to terminal or CI

Props

All props are optional. The component works with zero configuration.

onAnnotationAddFired when an annotation is created (receives Annotation)
onAnnotationDeleteFired when an annotation is deleted
onAnnotationUpdateFired when a note is edited
onAnnotationsClearFired when all annotations are cleared
onCopyFired when copy is clicked (receives markdown string)
onDrawFired per-stroke with shape detection result
enabledEnable/disable the component (default: true)
colorStroke color (default: "#171717")
strokeWidthStroke width in px (default: 3)
toolActive tool: "draw" | "arrow" | "circle" | "eraser"
detailOutput detail: "compact" | "standard" | "detailed" | "forensic"
toolbarShow floating toolbar (default: true)
position"bottom-right" | "bottom-left" | "top-right" | "top-left"
rootCSS selector for scan scope (default: "body")
ignoreArray of selectors to skip during element resolution
shortcutKeyboard shortcut (default: "ctrl+shift+d")
copyToClipboardAuto-copy on add (default: true)
endpointMCP server URL for syncing annotations
sessionIdPre-existing session ID to rejoin
onSessionCreatedFired when a new session is created (receives sessionId)

Basic usage

app.tsx
import { Markup, Annotation } from 'markupkit'

function App() {
  const handleAdd = (a: Annotation) => {
    console.log(a.shape.type, a.elements, a.note)
    // → "circle", [{selector: "div.mc", text: "Revenue"}], "fix height"
  }

  const handleDraw = (stroke) => {
    console.log(stroke.shape, stroke.confidence)
    // → "arrow", 0.92
  }

  return (
    <>
      <YourApp />
      <Markup
        onAnnotationAdd={handleAdd}
        onDraw={handleDraw}
        detail="detailed"
      />
    </>
  )
}

Types

The Annotation object passed to callbacks:

types.ts
type Annotation = {
  id: string                // Unique identifier
  note: string              // User's note text
  timestamp: number         // Unix timestamp (ms)

  shape: {
    type: 'circle' | 'arrow' | 'strikethrough'
          | 'underline' | 'cross' | 'rectangle' | 'freehand'
    confidence: number       // 0-1 shape detection score
    bounds: { x: number; y: number; width: number; height: number }
    head?: Point            // Arrow endpoint
    tail?: Point            // Arrow start
  }

  elements: {
    selector: string        // CSS selector path
    tag: string             // "button", "div", etc.
    text: string            // Text content (first 100 chars)
    component: string | null // React component name
    rect: { x; y; width; height }
    classes: string         // CSS classes
    id: string              // Element ID
  }[]

  intent: string            // "Circled 2 elements — area needs review"
}

type ShapeType = 'circle' | 'arrow' | 'strikethrough'
  | 'underline' | 'cross' | 'rectangle' | 'freehand'

type OutputDetail = 'compact' | 'standard' | 'detailed' | 'forensic'

type ToolMode = 'draw' | 'arrow' | 'circle' | 'eraser'

Real-time events

Subscribe to real-time events via Server-Sent Events:

terminal
# Session-level: events for a single page
curl -N http://localhost:4747/sessions/:id/events

# Global: events across all sessions
curl -N http://localhost:4747/events

# Filtered by domain
curl -N "http://localhost:4747/events?domain=localhost:3001"

# Reconnect after disconnect (replay missed events)
curl -N -H "Last-Event-ID: 42" http://localhost:4747/sessions/:id/events

Event types:

  • annotation.created — new annotation added
  • annotation.updated — annotation modified
  • annotation.deleted — annotation removed
  • stroke.created — new drawing stroke recorded
  • shape.detected — shape classified from stroke
  • element.resolved — DOM elements matched to shape
  • session.created — new session started
  • session.closed — session closed
  • thread.message — new message in annotation thread
  • design.scanned — full design DNA scan completed

Programmatic usage

Use the core functions without the React component:

script.ts
import {
  detectShape,
  resolveShape,
  formatSession,
  formatSingleAnnotation,
  smoothPoints,
  strokeBounds
} from 'markupkit'

// Detect shape from raw stroke points
const shape = detectShape(stroke)
// → { type: 'circle', confidence: 0.87, bounds: {...} }

// Resolve shape to DOM elements
const annotation = resolveShape(shape)
// → { elements: [{selector, text, component}], intent: "Circled 2 elements" }

// Generate markdown output
const md = formatSession(session, 'detailed')
// → "# MarkupKit Feedback\n3 annotations on..."
